分支管理

    # 列出本地分支
    git branch

    # 列出所有分支
    git branch -a

    # 列出远程分支
    git branch -r

    # 列出已经合并了的分支
    git branch --merged

    # 基于当前分支创建新分支
    git branch <branch-name>

    # 检出某个分支
    git checkout <branch-name>

标签管理

    # 列出标签
    git tag

    # （对当前位置）创建标签
    git tag v1.0

    # 检出标签，就像检出分支一样
    git checkout v1.0

提交

提交 是将暂存区的文件提交到仓库永久保存。 如果暂存区是空的，则不会提交。 

    git commit

    git commit -m "some message"

检出

检出 是将文件从仓库拿到工作区

    # 查看命令帮助
    git checkout -h

    # 分支检出
    git checkout master

    # 标签检出
    git checkout v1.0

    # 某个提交历史检出（commit id）
    git checkout d6a8b7069057bce6251757e62c206f94f36db9ff

    # 只检出某一个文件
    git checkout master a.txt  # 检出其他分支上的版本
    git checkout v1.0   a.txt  # 检出某个标签下的版本
    git checkout d6a8b7 a.txt  # 检出历史上的某一个版本

日志

日志 记录了提交的历史记录

    # 按时间的先后顺序列出所有的提交
    git log

    # 只显示最近的 5 次提交
    git log -5

    # 查看与前一版本的变化
    git log --patch

    # 查看简略统计信息（改动了哪些文件）
    git log --stat

    # 一行显示一次提交
    # --pretty=short
    # --pretty=full
    # --pretty=fuller
    git log --pretty=oneline

    # 相当于: --pretty=oneline --abbrev-commit
    git log --oneline

    # 树状图形展示
    git log --graph 
    
    # 组合使用
    git log --graph --pretty=oneline

日志的其他用法

    # 搜索改动中包含的字符串
    git log -S hello

    # 由于日志非常多，如果能指定目录或文件将能过滤掉很多无关的信息
    git log -- .     # 当前目录
    git log -- src   # 指定目录
    git log -- a.txt # 指定文件
